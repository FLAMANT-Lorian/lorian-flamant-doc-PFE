Validation du code HTML du site public

Afin de vérifier la sémantique du site public, j’ai utilisé l’outil « Total Validator ». Les erreurs visibles sur les captures d'écran ne sont pas vraiment de réelles erreurs de développement. Elles viennent soit de liens ou de styles générés automatiquement par WordPress. Sur la page d'archive des actualités, il y a également des erreurs générées par la pagination de WordPress, que je ne saurais pas corriger.

Validation du code HTML de l’intranet

Afin de vérifier la sémantique de l’intranet, j’ai également utilisé l’outil « Total Validator ». Étant donnée que j’utilise des Frameworks comme AplineJs ou encore Livewire, Total Validator détecte des erreurs qui n’en sont pas vraiment. Certains attributs comme x-data ou encore wire:navigate ne sont pas reconnus comme valide par le logiciels, alors qu'ils sont indispensables pour le bon fonctionnement du site.
