Tests automatisés

Méthodologie d’intégration

Lors de l’intégration des différentes fonctionnalités de l’intranet, j’ai mis en place un ensemble de tests automatisés afin de vérifier le bon fonctionnement des fonctionnalités réalisés. Ces tests permettent de m’assurer que les nouvelles fonctionnalités répondent aux exigences attendues par l’intranet, et qu’elles n’introduisent pas de bugs affectant les fonctionnalités déjà existantes. J'ai réalisé au total 120 tests différents, incluant des tests côté serveur, mais aussi d’interfaces à l’aide du Browser testing.

Présentation des tests côté serveur

Les tests côté serveur vérifient la logique métier de l’application sans passer par l’interface utilisateur. Ils permettent notamment de contrôler les validations, les permissions, les traitements de données, etc.
